## teoretical task 
```
1. Джанго смотрит введеный url пользователем и сводит его с нужным view

2. CBV - использует классы для определения view 
пример:
class TaskDetailView(DetailView):
    model = Task
    template_name = 'task_detail.html'

3. Модель - объект, описывающий данные в базе данных 
процесс:
1. объявление класса модели
2. описание его полей
3. описание его методов

регистрация в админке:
admin.site.register({Модель})

4. Формы обрабатывают введеные данные пользователем
валидация:
is_valid - метод валидации данных

ошибки:
{{ form.errors }} - отображение ошибок в шаблоне

5. Кастомные теги и фильтры, написанные пользователем
регистрация:
register = template.Library()

использование в шаблонах:
{% load {кастомный тег} %}
```
